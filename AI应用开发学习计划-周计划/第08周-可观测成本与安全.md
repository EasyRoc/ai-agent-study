# 第 8 周：可观测、成本、安全

> 约 **5～7 个学习日**。目标：**全链路**可关联一次请求（`requestId`/`traceId`）、**Token 与延迟** 可落日志或表、**超预算/超时** 有明确策略；**Guardrails** 做轻量**输入/输出**过滤（关键词或二次 LLM 拒答，选一种即可）；**记忆与合规**（**L4**）见 [`记忆架构-索引`](./记忆架构-索引.md) —— 对话/用户事实 **留存、删除、PII 不进日志**。  
> **栈延续**：在 Python 里若用 LangChain/LangGraph 的**回调**、**runnable 配置** 打 token/延迟，可与 FastAPI 中间件**同 requestId 关联**（**二选一**写清即可）；Java 侧以 **MDC/过滤器** 为主，**仍使用** `spring-ai-alibaba` 生态下的观测扩展（**若** BOM 中已带 starter，**按需** 打开，以官方说明为准）。

## 本周目标

- 定义统一天志字段（**JSON 一行一请求** 即可）：`ts`, `requestId`, `route`, `model`, `inputTokens`, `outputTokens`, `latencyMs`, `outcome=OK|ERROR`。
- Java：`HandlerInterceptor` 或 MDC 生成/传播 `X-Request-Id`；调 Python/模型前后打点。
- Python：在 FastAPI 中间件里同样打（若**仅** Java 对夕发请求，**至少**在 Python 从 Java 头里读 `X-Request-Id` 打印）。
- **预算演示**：**内存** 计数「当日某 API Key 已用 token」超阈值则**拒绝**并打 `BUDGET_EXCEEDED`（无真实账单也可）。
- **记忆与合规（必能讲）**：`session`/用户事实 存哪、**几天** 删、用户 **`DELETE`/`export`** 的**产品口径**；日志里**不** 完整打长对话，**只** 打 `hash`+长度+`requestId`。

## 按日计划

### 第 1 天：RequestId 贯通

- [ ] Java 入站若无 `X-Request-Id` 则生成 UUID，放入 MDC 与响应头。
- [ ] 调下游 HTTP 时**透传**该头；Python 日志里打印同一 id。

### 第 2 天：Token 与延迟

- [ ] 从模型返回里取 `usage`（各厂商字段名不同，**封装**一处）。
- [ ] 若取不到：用**字符数/4** 粗估，并在日志标 `ESTIMATED=true`（诚实说明面试）。

### 第 3 天：成本聚合（演示级）

- [ ] 按日 `Map<apiKey, totalTokens>`，暴露 `GET /admin/usage?day=today`（**无**真实鉴权也可，**勿**上公网裸奔）。
- [ ] 超 `DAILY_LIMIT` 时 `429` 或业务码 + `Retry-After`（可选）。

### 第 4 天：Guardrails 输入

- [ ] 简单 **正则/关键词** 拦截：明显违规词直接 `400` + 不打模型（**省成本**可讲）。
- [ ] 或：轻量**分类 prompt**（「是否违规 是/否」）仅对**灰区**长文本用（**控制调用次数**）。

### 第 5 天：Guardrails 输出

- [ ] 对模型回答做**后处理**：脱敏手机/身份证（**正则**即可演示）。
- [ ] 可选：若含「我帮你编写恶意」类输出，**模板拒答**替换为固定话术（规则 mock）。

### 第 6 天：超时与可重试

- [ ] 统一文档：`TIMEOUT` vs `MODEL_ERROR` vs `BUDGET_EXCEEDED` 各自**是否**建议用户重试、**是否**扣费（面试题准备）。

### 第 7 天：周文档

- [ ] 写 `docs/observability.md`：**字段表** + **示例** JSON 行 1 条（脱敏后）。
- [ ] 与第 7 周 `job` 的 `result` 里 **累计 token** 对一下口径，避免两套数字对不上（写一句说明以谁为准）。
- [ ] 写 `docs/privacy-memory.md` **半页内** 或 合并进 `observability.md` 一节：`memory` 相关**表** 或 集合、**PII 字段**、**留存/删除**、与 **L3** 的对应（**无** 实现可写「计划」）。

## 周产出物

- [ ] 统一日志样例 1 条 + `observability.md` 短文档。
- [ ] 一次「**超预算**」调用失败的录屏或日志片段（可本地）。
- [ ] 上述 **记忆/隐私** 半页（**可** 与 observability 合并）。

## 完成标准

- 能答：**可观测**第一解决的问题是「**钱与慢在哪**」还是「**错在哪一环节**」——你**两边都**有抓手；并能答**一句**：用户要求**删除记忆** 时，**L0/L2/L3** 各删什么（**Demo 级** 也可）。
