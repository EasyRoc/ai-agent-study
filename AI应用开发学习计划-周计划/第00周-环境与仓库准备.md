# 第 0 周：环境与仓库准备

> 预计 **2～3 天**（可穿插在正式第 1 周前周末完成）。不单独占满一周也可。

## 本周目标

- Python、Java 均可调用**同一**大模型 API，密钥不入代码库。
- 建立 **`monorepo-py`** 与 **`monorepo-java`** 两个子目录（见仓库根目录 [`README.md`](../README.md)），**Python / Java 代码分开放**，后续模块分别往里加。

## 时间分配


| 块   | 内容                                                    | 约时   |
| --- | ----------------------------------------------------- | ---- |
| A   | Python：**conda**（或 venv/uv）、IDE、`.env.example`     | 1～2h |
| B   | Java：JDK 17+、Spring Initializr、与 Python 同仓库根 `README` | 1～2h |
| C   | 模型账号与配额、**最小**聊天调用双端通                                 | 1～2h |


## 按日任务

### 第 1 天

- 安装 **Python 3.10+**。若使用 **conda**（推荐你当前习惯）：`conda create -n aimodel python=3.10 -y` → `conda activate aimodel` → `pip install openai`（或你选的厂商官方 SDK；亦可用 `conda install -c conda-forge openai` 等，以包是否齐全为准，**不冲突时 pip 在 conda 环境内装即可**）。若不用 conda：`python -m venv .venv` 或 `uv venv`，激活后同样 `pip install ...`。
- 在 `README` 里**写清** 环境名（如 `aimodel`）和激活**一行**命令，避免团队/日后自己换机时对不上号。
- 注册/申请 API Key，在 shell 中 `export` 或 `.env`（`.env` 已加入 `.gitignore`）。
- 在 **`monorepo-py/`** 中维护脚本：例如 `monorepo-py/scripts/health_chat.py`（**同步** 请求一次「你好」，打印回复前 200 字），具体步骤见该目录下 [`README.md`](../monorepo-py/README.md)。

### 第 2 天

- 在 **`monorepo-java/`** 下建 Spring 工程（如子目录 `java-demo`）：使用 **[Spring AI Alibaba](https://github.com/alibaba/spring-ai-alibaba)** 官方文档中的方式集成（**BOM、starter 名称、与 Spring Boot 3 版本** 以仓库当前文档为准，不必强行用 `start.spring.io` 若其未列出；可用 **空 Spring Initializr** + 再按官方 `pom.xml` 片段 引入依赖）。
- 本计划 **Java 侧** 以 **Spring AI Alibaba** 为**唯一**主线，与上册 [`AI应用开发学习计划-Java与Python双轨.md`](../AI应用开发学习计划-Java与Python双轨.md) 一致；概念上仍属 **Spring AI** 体系（`ChatClient`、advisor 等，具体类名**以你引入的版本** 为准）。
- 配置 `application.yml` 中**模型/鉴权** 项用**环境变量**引用（如 `DASHSCOPE_API_KEY` 或官方示例中的变量名），**不要** 写死 key。
- 写 `GET /api/v1/health/llm` 或等效接口，通过 **Spring AI Alibaba** 调一次**真实** 或 **mock** 模型回复，先保证链路通。

> **本仓库已实现参考**：[`monorepo-java/java-demo`](../../monorepo-java/java-demo/)（含 `pom.xml`、`HealthLlmController`、`application.yml` 与运行说明）；本地需 **JDK 17+**、**Maven**，并配置 `AI_DASHSCOPE_API_KEY` 或 `DASHSCOPE_API_KEY`。

### 第 3 天（或第 0 周结束日）

- 根目录 [`README.md`](../README.md) 中 **「第 0 周 · 第 3 天：双端收尾与验收」**（锚点 `week0-day3`）已写明：双目录、环境变量、启动顺序、`make health-py` / `make health` 等；子目录入口见同页表格。
- **`monorepo-py/.env.example`**：列出 `DASHSCOPE_API_KEY` / `OPENAI_*` / `LLM_MODEL` 等占位，**无真实值**；根目录另有 [`env.example`](../env.example) 作**全仓变量名**补充，**仍无**真密钥。
- 可选：仓库根目录 `Makefile` 中 `make health-py`、`make health-java`、`make health` 双端检查（**已提供**，需本机有 `make` 与 `curl`，Java 端需先起 `java-demo`）。

> 在 Cursor / 本地打开根目录 `README.md`，用 **Ctrl+F 搜「第 0 周 · 第 3 天」** 或锚点 `#week0-day3` 可直达该节。

## 常见坑

- 把 `API_KEY` 写进 `application.yml` 并误提交：务必用**环境变量** + `.gitignore`。
- 国内与海外 endpoint 不同：在 `README` 写死你当前使用的 `BASE_URL` 样例，避免来回改代码。
- **conda**：若同时装了 **多份 Python**，在 IDE 里**选对解释器**（指向该 env 的 `python`）；`conda` 与**系统 pip** 混用易装到错误环境，**先 `conda activate` 再 pip**。
- **Java / Spring AI Alibaba**：若需对接 **非默认** 厂商模型，**不要** 与另一套（如 LangChain4j）混在同一模块里**平行**挖两套文档；**优先** 看 Spring AI Alibaba 是否已提供**兼容/多模型** 配置，**否则** 用 **Python 子服务** 做该厂商网关，Java **仍只维护一条** 技术线。
