# 第 9 周：综合项目（双栈交付周）

> 约 **5～7 个学习日** + **1～2 天缓冲**。**本周是「能写进简历 + 能演示」的主交付。**

## 项目建议形态（在下列中选一种做深，**不要**全选）

1. **企业知识库问答**：上传 `Markdown`/`txt`（PDF 为选做），分块、索引、RAG+引用；可选轻量 Agent。  
2. **内部「制度/FAQ」小助手**：无上传，**预置**你手写的 20+ 条 `markdown` 当知识库。  

**必含**：Python 子服务 + Java 对外网关 + 至少一种 **流式**（SSE）入口 + 第 8 周级别的 **RequestId/usage 日志** + 根 `README` 一键/分步**可复现**。

**与记忆架构（必写）**：`docs/memory-architecture.md` 或 README 中专节——**L0 会话、L2 RAG 知识** 两条路径**必有**；**L3 用户事实** 若**未做** 须写**原因**；与 [`记忆架构-索引`](./记忆架构-索引.md) 分层**名词一致**。

**与第 6 周流行技术**（在下列中**至少**体现 **2 项**到 README「架构说明」中，**不必**全上代码）：

| 技术 | 在综合项目中的**可选/推荐**形态 |
|------|----------------------------------|
| **Function Calling** | 对外**或**内部 Agent 调工具时，**明确**使用 `tool_calls` / `@Tool` 路径（可与 **REST** 并存，**说明**谁用在哪）。 |
| **ReAct** | 至少**一条**用户路径走「多步工具」时，有**可保存**的轨迹（或**采样日志**）证明多步。 |
| **MCP** | **二选一**：① 第 6 周 MCP Server 升级为本项目**正式**工具面；② README 用半页写「**本可用 MCP 接**，当前 Demo 为 HTTP 直连」并给出**接法**草图。 |
| **Skills** | **二选一**：实现 **Intent → Skill 名 → 子工具子集** 的**简单路由**；或 在 README **链接** 第 6 周 `SKILL.md` 并说明**若上线**如何映射。 |

## 按日计划

### 第 1 天：需求冻结与切图

- [ ] 写 1 页 `SPEC.md`：**用户故事 3 条**（如「问制度」「带引用回答」「长报告异步」选其 2～3）；**显式**写是否包含 **MCP/Skills** 路由（**是/否** 一项即可避免扯皮）。
- [ ] 画**架构图**（Mermaid）：Browser → Java（认证/SSE/配额）→ Python（RAG/索引/**可选** Agent / **MCP 客户端?**）→ 模型；**若用 MCP**，增加 **MCP Server** 方框**或** 注明「**可选组件**」。
- [ ] 冻结**不做**范围（例如「不做多租户、不做 OAuth 级 MCP 鉴权」写清楚，防范围蔓延）。

### 第 2 天：Python 服务收口

- [ ] 单端口（如 `8000`）统一：`/ingest`（可简化）、`/search` 或 `/rag/ask`、**可选** `/agent/chat`、**健康检查** `GET /health`。
- [ ] 所有**慢操作**能走**第7周** job 的复用**或**明确写「本 Demo 仅同步、文档说明生产用 Kafka」。

### 第 3 天：Java 网关收口

- [ ] 路由：`/api/...` 全走 Java；仅 `/internal` 或直连禁止外网（可不做网络隔离，**文档**说明生产方案）。
- [ ] `SSE`：`/api/chat/stream` 转发到 Python 流式**或** Java 自接模型（**一条路径**说清即可）。

### 第 4 天：可观测与配置

- [ ] 环境变量：`.env.example` 列全 `BASE_URL`, `API_KEY`, `PY_VECTOR_URL` 等。
- [ ] 启动顺序：`README` 写「先起 Python 再起 Java」；失败时**常见 3 错**（端口、key、模型名）。

### 第 5 天：质量与「演示集」

- [ ] 准备 **5～10 个** 演示问句，覆盖：**命中知识**、**应拒答**、**需 Agent/工具**（若做了）。
- [ ] 修**明显** bug，不求完美。

### 第 6 天：录屏与仓库卫生

- [ ] **3 分钟**内录屏：clone → 配 env → 起服务 → 浏览器/**curl** 问 1 问（配字幕或**口头**可）。
- [ ] 删**临时** key、**大文件** 不入 git；`data/` 提供**小**样例可重建。

### 第 7 天：buffer

- [ ] 仅修阻塞演示的 issue；不新开大功能。
